## Replit Prompt (paste this)

You are a senior full-stack engineer + UI motion designer. I’m going to provide you an existing exchange website source code. Your job is to **use my existing codebase** (do not rebuild from scratch unless something is missing) and transform it into a polished product called **Prismatic** with an **Apple “Liquid Glass” theme** (glass blur, soft highlights, subtle glow, depth, smooth motion).

### 1) Project goals

* Website name: **Prismatic**
* Brand vibe: **greenish gradient gem / prism** (emerald → teal → lime, subtle spectral accents)
* Design style: **Apple Liquid Glass** like the attached reference image:

  * translucent frosted panels
  * background blur with saturation
  * soft inner highlights + subtle border
  * depth via layered glass cards
  * minimal, premium spacing
* Must feel smooth: **lots of animations**, but never cluttered or laggy.
* Must support:

  * **Live transactions feed** (real-time updates)
  * **Admin dashboard** to update content + upload icons/assets (PayPal, cards, crypto, etc.)
  * Ability to manage transactions shown on the homepage + admin side

### 2) Tech approach (follow this)

* Keep the current stack in my source code. If needed, upgrade UI layer with:

  * TailwindCSS (or keep current CSS system if already used)
  * Framer Motion (React) OR GSAP (if not React) for animations
* Implement a reusable **Glass UI system**:

  * `GlassCard`, `GlassPill`, `GlassModal`, `GlassButton`, `GlassInput`, `GlassNavbar`
* Add global theme tokens (CSS variables):

  * glass blur, border opacity, highlight strength, shadow depth, gradient palette
* Add a “Prismatic Gradient” background system:

  * slow moving gradient blobs + noise texture
  * optional mouse parallax on blobs
  * keep text readable with contrast layers

### 3) Liquid Glass design specs (must match)

Implement Apple-like glass using:

* Backdrop blur: `backdrop-filter: blur(18px) saturate(160%)`
* Glass surface:

  * background: `rgba(255,255,255,0.10)` (or dark-mode tuned equivalent)
  * border: `1px solid rgba(255,255,255,0.18)`
  * inner highlight (pseudo-element): soft top highlight gradient
  * shadow: subtle soft shadow, not harsh
* Rounded corners: large radius (18–28px)
* Pills/tabs like the reference image:

  * segmented control with glass pill container
  * selected item slightly brighter, smooth sliding indicator
* Typography:

  * Use **SF Pro / Inter** fallback (clean Apple feel)
  * Headings: tight letter spacing, clean hierarchy
  * Buttons: slightly rounded, glossy glass

### 4) Pages + layout (implement / upgrade)

**Public site**

* Landing page hero with Prismatic glass header + tabs/pills (Years/Months/All style)
* “Live Transactions” section:

  * real-time scrolling feed with payment method icon + amount + time + status
  * smooth entry animations (fade/slide)
  * filter pills: All / PayPal / Card / Crypto
* Exchange creation flow:

  * clean multi-step glass form
  * animated progress indicator
  * validation and success screen with glass confetti subtle effect

**Admin dashboard**

* Secure admin login (simple credential or token-based, depending on existing code)
* Admin features:

  * Upload/manage icons (PayPal, card, crypto, etc.)
  * Manage transaction feed data (add/edit/delete, toggle “featured”, change status)
  * Update site settings (brand text, gradient colors, announcements, min amount, fees)
  * Control animation intensity toggle (Low/Medium/High)
* Admin UI must also use the same glass design system.

### 5) Live transactions (important)

* If my current backend already exists, use it.
* If not, implement a minimal solution:

  * Store transactions in DB (SQLite/Postgres depending on project)
  * Provide API endpoints: list transactions, add transaction, update status
  * Real-time updates using **WebSockets** or **SSE** (Server Sent Events)
  * Frontend subscribes and updates feed instantly
* Include a “seed/demo mode” to generate fake transactions for testing (optional toggle).

### 6) Animations (lots, but premium)

Add motion that feels Apple-like:

* page transitions (fade + slight scale)
* hover: subtle lift + highlight sweep
* glass shimmer on buttons (very subtle)
* segmented control indicator slide
* list items animate in/out
* background blobs slowly drift
  Performance rules:
* prefer transform/opacity animations
* respect reduced-motion setting

### 7) Deliverables

* Update my repo with the new UI system + theme applied across the site
* Clean folder structure and reusable components
* Add a README with:

  * how to run
  * how to configure admin
  * where icons/assets are stored
  * how live transactions are generated/updated
* Do not leave placeholder content unfinished—make everything functional end-to-end.

Start by:

1. Reading my codebase structure and explaining what stack it uses.
2. Proposing the minimal changes needed.
3. Then implement the glass component system and apply it site-wide.
